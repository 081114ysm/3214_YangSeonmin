# DevFocus 요구사항명세서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | DevFocus - 개발자 학습 플랫폼 |
| 목적 | 개발자가 강의 수강, 집중 시간 관리, Q&A를 통해 효율적으로 학습할 수 있는 웹 플랫폼 |
| 기술 스택 | Frontend: Next.js 16, React 19, TailwindCSS / Backend: Express 5, MySQL 8, JWT |

---

## 2. 기능 요구사항

### 2.1 회원 관리 (Auth)

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|----------|
| AUTH-01 | 회원가입 | 이메일, 비밀번호, 닉네임으로 회원가입. 비밀번호는 bcrypt 해시 저장 | 필수 |
| AUTH-02 | 로그인 | 이메일/비밀번호 인증 후 JWT 토큰 발급 (유효기간 1시간) | 필수 |
| AUTH-03 | 로그아웃 | 클라이언트 토큰 삭제로 로그아웃 처리 | 필수 |
| AUTH-04 | 입력 검증 | 이메일 형식 확인, 비밀번호 4자 이상, 닉네임 2자 이상 | 필수 |
| AUTH-05 | 중복 검사 | 이메일 중복 가입 시 409 에러 반환 | 필수 |

### 2.2 사용자 정보 (Users)

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|----------|
| USER-01 | 내 정보 조회 | JWT 인증 후 로그인한 사용자의 id, email, nickname 반환 | 필수 |
| USER-02 | 닉네임 수정 | JWT 인증 후 닉네임 변경 (2자 이상 검증) | 필수 |

### 2.3 강의 관리 (Courses)

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|----------|
| COURSE-01 | 강의 목록 조회 | 전체 강의 목록 반환 (인증 불필요) | 필수 |
| COURSE-02 | 강의 상세 조회 | 강의 정보 + 소속 영상 목록 반환 (order 순 정렬) | 필수 |

### 2.4 학습 진도 (Progress)

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|----------|
| PROG-01 | 진도 저장 | 영상별 시청 시간 저장. 기존 기록이 있으면 업데이트 (UPSERT) | 필수 |
| PROG-02 | 진도 조회 | 특정 강의의 영상별 진도 조회 (영상 제목 JOIN) | 필수 |

### 2.5 집중 타이머 (Focus)

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|----------|
| FOCUS-01 | 집중 시작 | 새 세션 생성, 시작 시간 기록 | 필수 |
| FOCUS-02 | 집중 종료 | 종료 시간 기록, duration(초) 자동 계산 | 필수 |
| FOCUS-03 | 기록 조회 | 사용자의 전체 집중 기록 조회 (최신순 정렬) | 필수 |
| FOCUS-04 | 실시간 타이머 | 프론트엔드에서 경과 시간 실시간 표시 (HH:MM:SS) | 필수 |

### 2.6 Q&A 게시판 (QnA)

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|----------|
| QNA-01 | 질문 작성 | 제목(2자 이상) + 내용 입력, 인증 필수 | 필수 |
| QNA-02 | 질문 목록 | 전체 질문 목록 + 작성자 닉네임 (최신순 정렬) | 필수 |
| QNA-03 | 질문 상세 | 질문 내용 + 답변 목록 조회 | 필수 |
| QNA-04 | 답변 작성 | 특정 질문에 답변 등록, 인증 필수 | 필수 |

## 3. 비기능 요구사항

| ID | 항목 | 상세 설명 |
|----|------|-----------|
| NFR-01 | 인증/보안 | JWT 토큰 기반 인증, 비밀번호 bcrypt 해시 저장 |
| NFR-02 | CORS | 프론트엔드(localhost:3000)에서 백엔드(localhost:3001) 교차 출처 요청 허용 |
| NFR-03 | 입력 검증 | 프론트엔드 + 백엔드 양쪽에서 이중 검증 (클라이언트/서버 밸리데이션) |
| NFR-04 | 에러 처리 | 모든 API에 try-catch 적용, 적절한 HTTP 상태 코드 반환 |
| NFR-05 | 문자 인코딩 | DB utf8mb4 설정으로 한글/이모지 정상 처리 |
| NFR-06 | 데이터 무결성 | FK ON DELETE CASCADE, UNIQUE 제약조건 적용 |

## 4. API 명세

### 4.1 인증 API

| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| POST | /api/auth/register | X | 회원가입 |
| POST | /api/auth/login | X | 로그인 (JWT 발급) |

### 4.2 사용자 API

| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| GET | /api/users/me | O | 내 정보 조회 |
| PATCH | /api/users/me | O | 닉네임 수정 |

### 4.3 강의 API

| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| GET | /api/courses | X | 강의 목록 |
| GET | /api/courses/:courseId | X | 강의 상세 + 영상 목록 |

### 4.4 진도 API

| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| POST | /api/progress | O | 진도 저장 |
| GET | /api/progress/:courseId | O | 강의별 진도 조회 |

### 4.5 집중 타이머 API

| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| POST | /api/focus/start | O | 집중 시작 |
| POST | /api/focus/end | O | 집중 종료 |
| GET | /api/focus/history | O | 집중 기록 조회 |

### 4.6 Q&A API

| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| GET | /api/qna | X | 질문 목록 |
| POST | /api/qna | O | 질문 작성 |
| GET | /api/qna/:id | X | 질문 상세 + 답변 |
| POST | /api/qna/answer | O | 답변 작성 |

## 5. 화면 구성

| 페이지 | URL | 설명 |
|--------|-----|------|
| 홈 | / | 히어로 섹션, 추천 강의, 기능 소개 |
| 로그인 | /login | 이메일/비밀번호 입력, 입력 검증 |
| 회원가입 | /register | 닉네임/이메일/비밀번호 입력, 입력 검증 |
| 대시보드 | /dashboard | 총 집중 시간, 세션 수, 최근 기록 (인증 필수) |
| 강의 목록 | /courses | 전체 강의 카드 목록 |
| 강의 상세 | /course/:id | 강의 정보 + 영상 목록 (시간 표시) |
| 집중 타이머 | /focus | 실시간 타이머 + 기록 히스토리 |
| Q&A | /qna | 질문 작성/목록/상세/답변 |

## 6. 밸리데이션 규칙

### 서버 (Backend)
| 항목 | 규칙 |
|------|------|
| 이메일 | 필수, 이메일 형식 정규식 검증 |
| 비밀번호 | 필수, 4자 이상 |
| 닉네임 | 필수, 2자 이상 |
| 질문 제목 | 필수, 2자 이상 |
| 질문/답변 내용 | 필수, 빈 값 불가 |
| 진도 저장 | lessonId 필수, watchedSeconds 0 이상 |

### 클라이언트 (Frontend)
| 항목 | 규칙 |
|------|------|
| 이메일 | 필수, 이메일 형식 정규식 검증 |
| 비밀번호 | 필수, 4자 이상 |
| 닉네임 | 필수, 2자 이상 |
| 질문 제목 | 필수, 2자 이상 |
| 질문/답변 내용 | 필수, 빈 값 불가 |

> 클라이언트와 서버 양쪽 모두 동일한 규칙으로 이중 검증 적용
